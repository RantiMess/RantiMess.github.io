vstSmdIO -h;
// 
 vsSmdIO - Valve Source Smd Input/Output

 Command to export and import data to and from Valve SMD (VRT) files.
  The Valve SMD File Translator is merely a front end to this command which
 is always used to do the actual work.

 Command Line Switches:

 -h    | -help  . . . . . . . Print this information

 -f    | -filename  . . . . . On both import and export, use the specified filename

 -i    | -import  . . . . . . Import an SMD file from the file specified by -filename
 -o    | -origin <x> <y> <z>  When importing, translate the resulting model by the specified position
 -a    | -angles <x> <y> <z>  When importing, rotate the resulting model by the specified degree angles
 -vmf  | -valveMapFile . . .  Tell vstSmdIO that the import is to go along with a Valve Map file so the orientations are different
 -t    | -textureArchive <$>  Archive all file textures into the specified directory
 -is   | -importSkeleton <?>  Determines whether the skeleton will be imported.  By default only the model mesh is imported
 -v2t  | -vsVmtToTex <!>  . . By default, vsVmtToTex nodes will be used to assign materials if possible.  This introduces a
                              dependency on the valveSource tools.  Specify -v2t false to turn off and only create maya nodes
 -it   | -importType  . . . . What sort of data to import.  One of:
                              reference ( Model )              r, m
                              animation ( Skeletal Animation ) a, s
                              NOTE: Defaults to model.  Unlike export type, only 1 import type can be specified
 -eh   | -edgeHardness  . . . Sets the edge hardness on the model.  Note, this is a slow operation

 -e    | -export  . . . . . . Export Maya data to the DMX file specified by -filename
 -et   | -exportType <?>  . . What sort of data to export.  One of:
                              reference   ( Model )              r, m
                              physbox     ( Physics Model )      p
                              animation   ( Skeletal Animation ) a, s
                              vta         ( Vertex Animation )   v
                              vertexCache ( Animated Vertices )  vc
                              NOTE: vc imply -bc
                              NOTE: Only the shortest unique sequence of characters to
                                    uniquely identify the type is necessary. e.g. -et a.
                                    The list of acceptable characters is listed at the end of each type.
                                    To get both a model and its animation in the same smd, specify
                                    -exportType twice, i.e. -et m -et a
 -fs   | -frameStart <#> . . Specify the start time for animation export.  If not specified, the current active
                             animation start time for the scene will be used
 -fe   | -frameEnd <#> . . . Specify the end range for animation export.  If not specified, the current active
                             animation end time for the scene will be used
 -fi   | -frameIncrement <#> Specify the time increment for animation export.  If not specified, the current
                             active animation increment time for the scene will be used
 -sl   | -selection  . . . . Only export selected objects in the scene
 -up   | -upAxis <?> . . . . Export the data from Maya as if it oriented with the specified up axis,
                             one of X, Y or Z.  If not specified the default will be Y.  No matter what
                             the Maya up axis is set to, if you do -upAxis ? there needs to be a corresponding:

                             $upaxis ?

                             command in the .qc file.  Where ? is X, Y or Z.  Note that case is unimportant in
                             this tool and in .qc, i.e. x, y or z work
 -ds   | -doubleSided  . . . If specified, all meshes in the scene which have .doubleSided true will be output
                             twice.  NOTE: Use with caution as .doubleSided defaults to true on Maya meshes
 -w    | -world  . . . . . . Collapse the bone hierarchy into world space
 -rp   | -removePrefix <$> . Removes the specified pre-existing node prefix before adding any specified node
                             prefix via -np on import or export
 -np   | -nodePrefix <$> . . Prefix each node with the specified string.  By default the Maya node name is
                             used directly in the smd file.  If, for compatibility reasons, each node needs
                             a prefix this command switch can be used, e.g. -nodePrefix "ValveBiped."
                             This works on import and export and occurs after any -rp command -ei   | -exportInvisible  . Normally invisible, intermediate & template nodes are ignored, specify this flag
                             to cause them to be exported as well.
 -rm   | -relativeMaterials  Use relative material paths.  Note that this requires a version of studiomdl
                             that came out of Valve main after June 14, 2006.  Use of this flag
                             makes $cdmaterials qc commands unnecessary -pp   | -prettyPrint <!>  . By default SMD's will be pretty printed.  Specify -prettyPrint false to
                             revert back to old style
 -qci  | -qciFile <$>  . . . Export extra data such as attachments, etc... into the specified qci file
                             If -exportType is model then attachments, etc.. are output to the qci
                             If -exportType is vta then flex names, etc.. are output to the qci
 -bc   | -bindCurrent  . . . Use the current pose of the skeleton and mesh as the bind pose
 -sam  | -shaderAsMaterial . Use the name of the surface shader node as the material name

 Automation Options:

 -m    | -materials  . . . . Do all of -materialSrc, -materialVtf, -materialVmt
 -src  | -materialSrc  . . . Copy any textures used on the exported model
 -vtf  | -materialVtf  . . . Run vtex on the textures to convert them to vtf files
 -vmt  | -materialVmt  . . . Output a simple vmt file (VertexLitGeneric with the texture as $basetexture) for each texture mapped material used
 -qc   | -qcCreate . . . . . Create a simple qc file for compiling the model
 -qcl  | -qcCollapseBones  . Output $collapsebones command in qc
 -qce  | -qcCenterBonesOnVerts Output $centerbonesonverts command in qc
 -qsb  | -qcSkipBonesInBBox  Output $skipbonesinbbox command in qc

 NOTE: If any automation options are set the result returned from the command will
       be a string array of pairs of strings, type and the pathname.
